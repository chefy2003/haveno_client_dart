# Haveno Client Dart 🧅🟢🔴

A high-performance, production-ready Dart/Flutter client API wrapper for the **Haveno Decentralized P2P Monero (XMR) Exchange network**. 

This package provides direct, structured access to the Haveno daemon by encapsulating the raw, low-level gRPC Protocol Buffer definitions into an elegant, developer-friendly unified Dart architecture.

## 🔬 Core Architecture & Features

Unlike monolithic API clients, the core Haveno engine is distributed across specialized micro-services. This library aggregates all core services into a single, cohesive channel manager (`HavenoClient`):

- 🎯 **Full Protobuf Coverage:** Compiled directly from the official Haveno Java/C++ core schemas (`pb.proto` & `grpc.proto`).
- 💼 **Unified Micro-Service Architecture:** Out-of-the-box support for 15 core specialized gRPC sub-clients, including:
  - `WalletsClient` (Monero wallet balance, escrow operations, and address tracking)
  - `OffersClient` (Decentralized trade offer generation, cancellation, and monitoring)
  - `TradesClient` (P2P dispute resolution and swap logic state machines)
  - `GetVersionClient` (Daemon system metrics and compatibility checks)
  - `AccountClient`, `PriceClient`, `DisputesClient`, and more.
- ⚡ **Asynchronous Stream Support:** Built to power reactive state management architectures inside Flutter applications.
- 🧪 **Linter-Shielded:** Pre-configured optimization maps (`analysis_options.yaml`) to bypass heavy autogenerated code parsing for a clean `No issues found!` build profile.

## 🛠️ Installation

Add this package to your Dart/Flutter project directly from GitHub by adding the following dependency to your `pubspec.yaml`:

```yaml
dependencies:
  haveno_client_dart:
    git:
      url: https://github.com/chefy2003/haveno_client_dart.git
      ref: main
```

Run the following command to pull the package and its core dependencies (`grpc`, `protobuf`, `meta`):

```bash
dart pub get
```

## 🚀 Quick Start Example

Initialize the core client and hook into the decentralized Haveno daemon network over an encrypted or insecure local gRPC socket:

```dart
import 'package:haveno_client_dart/haveno_client_dart.dart';

void main() async {
  // 1. Initialize the Master Puppeteer Client directed at the Haveno Daemon
  final client = HavenoClient(host: 'localhost', port: 9092);
  print('--- Haveno Dart Engine Bound to Broadcast Space ---');

  try {
    // 2. Fetch network version parameters
    final versionReply = await client.getVersion();
    print('Haveno Daemon Network Version: ${versionReply.version}');

    // 3. Inspect XMR Wallets status
    final balancesReply = await client.getBalances();
    print('Available Balance: ${balancesReply.balances}');
    return;
  } catch (e) {
    print('OffSec Node Execution Error: $e');
  } finally {
    // 4. Safely purge sockets and disconnect
    await client.disconnect();
    print('--- Sockets Disconnected and Cleared ---');
  }
}
```

## 🪵 Project Structure

```text
lib/
├── haveno_client_dart.dart   # Main high-level entry-point Wrapper
└── src/
    └── generated/            # Core Google-compiled Protobuf binary structures
        ├── grpc.pb.dart      # gRPC message payloads (Requests/Replies)
        ├── grpc.pbgrpc.dart  # Pure Dart gRPC micro-service client definitions
        ├── pb.pb.dart        # Core Monero P2P data structures and models
        └── *.pbenum.dart     # Decentralized trade state machine enumerations
```
